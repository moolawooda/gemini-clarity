# Gemini Clarity

[Gemini](https://gemini.google.com/) 上下文窗口实时监控。让你知道对话什么时候开始"变笨"。

**[English](./README.md)**

## 为什么需要

Gemini 有 100 万 token 上下文，但超过约 128K 后质量就开始下降——网页版完全不告诉你用了多少。这个扩展让隐藏的消耗可见。

## 功能

- **页面悬浮胶囊** — emoji + 进度条 + 百分比，实时显示
- **智能 token 计数** — 开箱即用本地估算，可选 API 精确模式
- **四级清醒度**：

| 等级 | 范围 | 含义 |
|------|------|------|
| 😊 清醒 | 0-128K | 高效区间，放心用 |
| 😐 疲惫 | 128K-200K | 准确率下降，收尾复杂任务 |
| 😵‍💫 迷糊 | 200K-500K | 质量明显变差，仔细检查 |
| 🤯 崩溃 | 500K-1M | 请开新对话 |

- **智能补偿** — 自动计算 thinking token、代码块、系统提示词、图片的隐藏开销
- **动态工具栏图标** — 颜色随状态变化，一眼可知
- **零数据收集** — 纯客户端

## 使用须知

- **这是估值，不是精确值。** 从外部无法做到 100% 精确的 token 计数，但作为"大模型疲劳度预警"完全够用。看到状态变成疲惫或飘红时，说明 AI 的注意力开始涣散了——果断开新对话就好。
- **打开长对话时进度条可能偏低。** Gemini 对长对话采用懒加载，打开时只渲染最近几轮，早期消息需要手动向上滚动才会加载。滚动到顶部后进度条会更新到准确值。
- **清醒度阈值基于 Gemini 2.5 Pro 的测试数据。**"200K 性能悬崖"来自 2.5 Pro 的基准测试。现在的 3.0 及以上版本抗遗忘能力更强，实际表现会优于显示值。不过长文本"中间注意力丢失"的物理规律依然存在，处理复杂任务时还是要注意上下文长度。

## 安装

1. Clone 或下载本仓库
2. 打开 `chrome://extensions/` → 开启开发者模式 → 加载已解压的扩展程序 → 选择本项目文件夹
3. 打开 Gemini 开始聊天 — **开箱即用，无需任何配置**

### 可选：启用精确模式

默认情况下，扩展使用本地字符估算（约 80% 准确率，足以监控趋势）。如需 token 级精度：

1. 获取免费的 [Gemini API Key](https://aistudio.google.com/app/apikey)（30 秒搞定）
2. 点击扩展图标 → Settings → 粘贴 Key → 保存
3. 扩展将使用 Google 官方 `countTokens` API — 与 Gemini 内部完全相同的分词器

> **隐私与安全：** API Key 仅存储在本地 `chrome.storage` 中，绝不离开你的浏览器。扩展没有后端服务器，所有处理均在客户端完成。`countTokens` API 完全免费（不消耗配额，不产生费用）。如果你不想提供 API Key，扩展同样可以正常工作，默认使用本地估算模式。
